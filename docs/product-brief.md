# Product Brief

## Working Name

Maple

## One-Line Pitch

Use the AI subscription you already pay for to turn source material into a living, source-grounded wiki.

## Target User

Students, self-learners, personal archivists, and small teams who:

- already pay for ChatGPT or are willing to use it
- have PDFs, lecture notes, papers, web clips, screenshots, or personal notes
- want organized understanding, not just a temporary chat answer
- are not comfortable with open source tooling, Obsidian configuration, terminal commands, or agent setup

## Product Positioning

This should not be positioned as:

- a generic note app
- an Obsidian replacement
- a PDF chatbot
- a course-only app

It should be positioned as:

> A wiki workspace that compiles your sources into a maintained knowledge base.

## Core Difference

Most AI learning tools answer questions from documents. This app should maintain a compounding knowledge base:

- sources stay immutable
- summaries capture each source
- concept pages become canonical explanations
- guides create routes
- logs show what changed
- users can see what changed and undo the last AI write operation

## First Wedge

Start with college students and self-learners because their pain is urgent and easy to demonstrate, while keeping the workspace model broad enough for personal archives and small firm knowledge:

- scattered PDFs and slides
- lecture notes
- exam prep
- project research
- repeated questions across related material

The architecture should stay generic: a workspace can represent a course, research topic, project, book, exam, or skill path.
